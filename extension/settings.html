Using local Ollama? Fix the 403 Forbidden first

By default Ollama refuses requests from a browser extension, so ChatPanel gets 403 Forbidden. Start it so it accepts extension origins: 

OLLAMA_ORIGINS="chrome-extension://*" ollama serve

This allows browser extensions only — safer than "*", which would let any website you visit reach your local Ollama. To lock it down further, use this extension's exact id from chrome://extensions: OLLAMA_ORIGINS="chrome-extension://<id>".

Keep that terminal running, then set Base URL http://localhost:11434/v1 and click Load models. The API key can be left blank.

Web search — fetch live results as context for any model

Type /search <query> (anywhere in a message) to fetch web results and attach them as context — no tabs open, works with every model/agent. Engines use a Chrome-style %s (or {q}) query placeholder and must return server-rendered HTML to a fetch (Google's main page doesn't, so it's off by default).

Smart suggestions — page-specific prompt ideas on an empty chat

A new chat shows quick prompt ideas. With this on, ChatPanel tailors them to the page you're viewing by sending only lightweight metadata — the URL, title, and description (never the page body) — to your configured model. Off by default, so simply browsing never triggers a model call; enable it once to use it on every site. When off (or if the call fails) you get the universal fallbacks.

MCP tool servers — external tools for your agents

Connect Streamable HTTP MCP servers; their tools become callable by your API agents (alongside “Act on page”). The extension can’t run stdio servers directly — front those with an HTTP endpoint (e.g. mcp-proxy). Free includes one MCP server; Pro is unlimited.

Discover — official registry and curated public servers

Search the official MCP registry, or one-click add a curated public server. Endpoints & availability are set by their providers and can change. Ones marked auth need a token/login you add after.

Skills

Reusable prompts you drop into any chat with the button. Editing the built-ins is free; creating your own is Pro.

Meetings — live scribe for Zoom, Meet, Teams & Webex

Pro

On a supported call, ChatPanel turns the live captions into an auto-refreshing summary and a searchable transcript you can ask about mid-meeting. Captions only — never audio or video. These settings control how the scribe behaves.

Concise keeps the running summary short and cheap (recommended for long calls); Detailed produces full minutes. You can always regenerate either style per meeting in the meetings dashboard.

Meeting monitors: turn any of your Skills into a one-tap live monitor with “Use as a live-meeting monitor”, then launch it (or /monitor / /tldr) from the panel above the composer during a call.

Auto-refresh merges new captions into the running summary on that interval and saves it to Past Meetings automatically. A shorter transcript window keeps long meetings from growing the prompt (and cost) on every question — use the full transcript for short calls, a window for long ones.

Notes — your private, on-device writing workspace

Notes are encrypted on your device and searchable alongside your chats and meetings. These preferences apply to the notes editor and its AI commands.

Co-writer — a small model team that drafts, fixes & researches as you write

When on, an editor fixes typos/grammar on a pause, a researcher surfaces related material, and a writer drafts ahead on ⌘↵. Off by default — it makes model calls as you type. Pick a model per role from the team button in the note editor's footer.

Inline autocomplete — as-you-type ghost suggestions, Tab to accept

On a short typing pause at the end of a note, a model predicts the next few words and shows them as grey ghost text — press Tab to accept, Esc or keep typing to dismiss. Off by default (it calls a model as you write). Accepted text is attributed to the model in the note's history.

Note commands — @insert, @table, @list, @summarize, @translate

Type @insert <instruction> on a line to generate or fetch content and drop it into the note. Commands use your active ChatPanel model and run in the background — switch notes freely while they work.

Tools these commands may use — uncheck to limit notes to the model's own knowledge; checked follows your global ChatPanel setting.

Content is redacted per your Privacy settings before it leaves your device.

Backup & storage — notes ride your ChatPanel backup

Notes are included in your encrypted export and (Pro) daily auto-backup to disk — configured in the Account tab.

Chat history — search every conversation

Browse and full-text search every chat in a full-screen dashboard — ranked relevance (BM25), keyword and by-agent search, the conversation thread, related chats and a relationship graph. Everything stays on your device.

Privacy — redact PII before it reaches a model

Sensitive values are replaced before anything reaches a model, then restored in the reply you see. Patterns catch structured secrets (emails, phones, cards, keys, IPs) + your dictionary; AI detection (Pro) adds names, orgs & locations.

# macOS / Linux
curl -fsSL https://dl.chatpanel.net/gateway/install.sh | bash

# Windows (PowerShell)
irm https://dl.chatpanel.net/gateway/install.ps1 | iex

# or via npm — any OS, needs Node
npm i -g @chatpanel/gateway && chatpanel-gateway --install

2 · Add it as a model — open the API tab and create a custom OpenAI-compatible endpoint pointing at the gateway:

Base URL : http://127.0.0.1:4320/v1
API key  : local            (any value — the gateway forwards your real key / login)

Preferences

Runs after chats or meeting transcripts change, then saves concise graph topics locally. Blank uses the active model/agent.

As you pause typing, a model suggests a short continuation — press Tab to accept. For this to feel instant it needs a small, fast API model (e.g. a 1–3B model on your local Ollama / llama.cpp) — set it as an endpoint's “Autocomplete model” below. Local CLI agents spawn a process per request (~seconds), so they’re too slow for inline autocomplete. Avoid reasoning models — they “think” instead of completing.

When Act on page is on, ask for your OK before the agent fills a form, clicks, or types — so a web page, meeting transcript, or connected tool can’t quietly steer it into acting for you. Pick Allow for this site to stop being asked on a site you trust. On by default.

When the agent fills forms or clicks ( Act on page), use trusted browser events instead of synthetic ones — far more reliable on heavy web apps like Google Flights. Chrome shows a “ChatPanel is debugging this browser” banner while it’s active. Off by default. API agents only.

Heads up: acting on the page is best-effort automation driven by an AI model — it can pick the wrong field or misread a layout, and content it reads (web pages, meeting transcripts, connected tools) can try to influence it. Keep Confirm before page actions on for anything that matters and review each action before you allow it — especially on banking, email, or work accounts. You stay in control: ChatPanel only acts when you turn this on and ask.

Back up & restore your data Pro

Export everything to a single file — your settings (API endpoints & keys, agents, MCP servers, skills, preferences), endpoint sign-ins, all chat history and all captured meetings. Restore it on a fresh install on another machine to bring your whole setup across. Everything stays on your device — nothing is uploaded. Your Pro license isn’t included — it re-activates from your purchase email.

With no password you get a browsable .zip (the restorable backup plus readable Markdown copies of every chat and meeting) — it contains your API keys, so keep it safe. Set a password and you instead get a single encrypted .json file (no Markdown — that’s the point); you’ll need the same password to restore it.

Set a password to encrypt the export; you’ll need the same password to restore it. There’s no recovery — if you forget it the data is gone. Leave blank to export the plain browsable .zip. To restore an encrypted file, type its password here first, then choose the file.

Automatic daily backup to disk

Keep a copy of all your data outside the extension so a reinstall can never lose it. Once a day (when something changed) ChatPanel writes the full backup .zip to your Downloads → ChatPanel Backups folder, rotating by weekday so the last 7 days are kept. If you ever reinstall and the extension loses its data, just Restore from file with the newest backup. The file contains your API keys & sign-ins — it stays on your disk and is never uploaded, so keep your Downloads folder safe.

Pick a time of day (your local time), or leave it automatic. “Back up now” below is a one-off and doesn’t change this schedule.

Encrypts every daily file so a backup that leaves this machine (cloud-synced Downloads, a shared PC) is unreadable without the password. Because the backup runs unattended, this password is stored locally on this device. Restore it the same way — type the password in the box above before choosing the file. Turning encryption on automatically removes any earlier plain .zip backups ChatPanel wrote, so no unencrypted copy is left behind.
