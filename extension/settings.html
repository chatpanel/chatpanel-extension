Using local Ollama? Fix the 403 Forbidden first

By default Ollama refuses requests from a browser extension, so ChatPanel gets 403 Forbidden. Start it so it accepts the extension's origin: 

OLLAMA_ORIGINS="*" ollama serve

Keep that terminal running, then set Base URL http://localhost:11434/v1 and click ↻ Load models. The API key can be left blank.

Preferences

As you pause typing, a model suggests a short continuation — press Tab to accept. For this to feel instant it needs a small, fast API model (e.g. a 1–3B model on your local Ollama / llama.cpp) — set it as an endpoint's “Autocomplete model” below. Local CLI agents (Claude Code, Codex, Antigravity…) spawn a process per request (~seconds), so they’re too slow for inline autocomplete and are only used as a last resort. Avoid reasoning models — they “think” instead of completing.

When the agent fills forms or clicks (🖋 Act on page), use trusted browser events instead of synthetic ones — far more reliable on heavy web apps like Google Flights. Chrome shows a “ChatPanel is debugging this browser” banner while it’s active. Off by default. API agents only.

Heads up: acting on the page is best-effort automation driven by an AI model — it can pick the wrong field, choose the wrong option, or misread a layout. Review before you submit anything that matters. ChatPanel only clicks and types when you ask it to.
